## 2. 核心叙事重构

### 2.1 语言翻译矩阵

| 技术语言 | ESG 语言 | 用途 |
|---------|---------|------|
| 机器可读的 YAML spec | 结构化合规规则库 | 对外宣传 |
| JsonLogic 表达式 | 自动化合规校验 | 对外宣传 |
| External Functions | 复杂场景计算引擎 | 技术文档 |
| 双层架构 | 规则 + 知识库分离 | 架构说明 |
| AI Agent 消费 | AI 辅助碳核算 | 对外宣传 |
| fallback_chains | 数据质量优先级链 | 对外宣传 |
| require_justification | Comply or Explain 机制 | 行业沟通 |

### 2.2 一句话定位

**给 ESG 从业者：**
> 这是一套把 GHG Protocol Scope 2 规则转化为结构化数据的开源工具，让 AI 能自动校验碳核算结果是否合规——每条判定都能追溯到标准原文。

**给 AI 开发者：**
> 机器可读的 GHG Protocol Scope 2 spec，用 JsonLogic + External Functions 实现，任何 AI agent 都能直接加载执行。

### 2.3 核心叙事框架

```
痛点：碳核算是合规性工作，AI 算完没人知道对不对
问题：GHG Protocol 是 PDF，机器读不懂
方案：我们把它变成了结构化规则
价值：每条判定可追溯，每步计算可审计
```

## 5. GitHub 仓库优化

### 5.1 README 重写

**目标：** 让 ESG 从业者 30 秒内理解这个项目是什么、能做什么。

**新结构：**

```markdown
# GHG Protocol Scope 2 — 结构化合规规则库

> 把 GHG Protocol 的 Scope 2 规则转化为 AI 可消费的结构化数据。
> 每条规则可追溯，每步计算可审计。

## 这是什么？

一套开源的 YAML 规则文件，定义了 GHG Protocol Scope 2 的所有合规要求。
任何 AI agent 或碳核算软件都可以直接加载这套规则，自动校验核算结果。

## 它能做什么？

✅ 校验核算数据是否完整
✅ 自动选择最优排放因子（含数据回退链）
✅ 检测 Scope 1/2 边界重叠
✅ 强制双重报告合规
✅ 每条判定追溯到 GHG Protocol 原文

## 它不能做什么？

❌ 不是碳计算器（不含排放因子数据库）
❌ 不替代 GHG Protocol 标准本身
```

## 6. LinkedIn 发帖策略

### 6.1 发帖时间

- **最佳：** 工作日早上 8-9 点（ESG 从业者刷 LinkedIn 的时间）
- **次佳：** 周二/周三
- **避免：** 周末、晚上

### 6.2 发帖格式

**短文（300-500 字）：**
- 3-5 个短段落，每段 1-2 句
- 开头用问题或数据引起注意
- 结尾加 CTA（Call to Action）

**长文（1000-1500 字）：**
- 加小标题，每段不超过 3 句
- 用粗体标注关键信息
- 结尾加"欢迎讨论"
